## Still Open

- Full modern backtest integration is not done. The repo now has a stable adapter boundary, but replay still runs through legacy logic rather than the modern `ContextService` / `ArchitectService` / `TradingBot` path.
- Logging cleanup is still open after the recent trimming patches; reduce noise without dropping structured risk/telemetry fields.
- Launcher preparation is still open:
  - startup needs an explicit `Normal` / `Debug` mode selector
  - `Debug` mode should drive a second configuration window for run-capture selection
  - the selected capture set should map cleanly onto `jsonl` output without baking debug concerns into trading logic
- Debug-output schema is still open:
  - decide which records are append-only event lines
  - decide which values are rolling counters or latest snapshots that should update by number instead of exploding log volume
- Short support is no longer just prep work, but it still needs an end-to-end audit across replay, analytics, and operator flows before it should be treated as fully settled.
- Pulse startup auto-open still depends on the default OS/browser opener; window sizing and browser-specific launch behavior remain intentionally non-guaranteed.
- MTF is enabled by default in `src/data/bots.config.json`; use `MTF_ENABLED=false` to disable it at runtime or `MTF_ENABLED=true` to force it on if JSON config is changed.
- MTF diagnostics are observable, but there is no UI control surface for MTF tuning.

## Recommended Next Order

1. Clean up the current logging noise while preserving structured telemetry, risk signals, and test coverage.
2. Define the launcher startup contract:
   - first window selects `Normal` vs `Debug`
   - `Debug` opens a second window for selecting run-capture fields
   - keep the launcher/config surface separate from trading decisions
3. Define the debug capture schema before implementation:
   - append-only `jsonl` event families
   - rolling numeric metrics that should be stored as latest values or aggregated counters
   - naming and stability rules for fields that external tooling may consume
4. Extend `BacktestEngine` from adapter-only into a replay feeder that can drive modern decision components without copying legacy runtime logic back into `src/`.
5. Continue short-readiness work as an audit pass, not a greenfield feature, by re-checking:
   - `src/roles/exitDecisionCoordinator.ts`
   - `src/bots/tradingBot.ts`
   - `src/core/systemServer.ts`
   - strategy modules under `src/strategies/`
6. If more performance work is needed, stay local to `ContextBuilder`, `ContextService`, and data-shaping utilities. Avoid broad runtime rewrites.
7. If operator tooling is expanded, keep it inside Pulse and on top of the explicit manual-resume API; do not infer resume semantics from generic state changes.
8. Keep exit capability semantics authoritative if launcher/debug work surfaces them:
   - disabled capability flags must not still cause exit or downstream reclassification
9. Preserve paused-state coherence:
   - no persisted `paused + null reason`
   - generic pause reasons remain runtime-authoritative for entry suppression
   - only `max_drawdown_reached` remains resumable through the explicit server API
10. If Architect churn still appears in paper runs, instrument publication count since entry and invalidation grace decisions before changing strategy formulas.
11. If MTF tuning continues, keep changes local to MTF config/aggregation/resolver/economics diagnostics and preserve baseline behavior when `mtf.enabled=false` or `MTF_ENABLED=false`.

## Do Not Do Next

- Do not re-open the live execution path from the active runtime.
- Do not treat `BacktestEngine` adapter coverage as proof of replay parity.
- Do not move architect, latch, or outcome-shaping logic back into `TradingBot`.
- Do not assume short support is "done" just because runtime paper tests pass; replay/reporting/operator flows still need explicit confirmation.
- Do not reintroduce single-publish `family_mismatch` invalidation for managed recovery.
- Do not reintroduce a second dashboard architecture beside Pulse.
- Do not let launcher mode selection or debug-output capture leak into runtime trading decisions.
- Do not serialize every tick-sized detail into `jsonl` by default when a rolling numeric snapshot or counter would carry the same signal more cheaply.
- Do not move RSI target-distance rejection into the strategy formula.
- Do not put MTF interpretation or strategy-specific MTF branches into `TradingBot`.
- Do not let raw timeframe labels leak into `mtfParamResolver`.
