# Session Handoff — 2026-04-10

## Completed

- Phase 1: `StateStore` now tracks symbol touch times, evicts stale symbol-scoped state conservatively, and exposes cleanup diagnostics. `ContextService` can prune its symbol-local memoization after eviction.
- Phase 2: REST fallback now skips fresh symbols, narrows requests to stale symbols, and uses `fetchTickers(...)` batching when multiple symbols need refresh.
- Phase 3: `src/engines/backtestEngine.ts` is no longer a scaffold. It is now a modern adapter boundary over the preserved legacy backtest modules.
- Phase 4: `ContextBuilder` hot-path allocations were reduced by replacing several repeated `map` / `slice` remaps with local loops. `RegimeDetector` did not need a new patch because the earlier slice-heavy issue is already gone in the current code.
- Phase 5: trade-side typing and directional PnL helpers are now side-ready for future short work without enabling short entries or changing current paper-long behavior.
- Follow-up telemetry pass: the older `TradingBot` max-drawdown manual-resume regression is fixed. Manual-resume-required drawdown pauses now emit explicit `RISK_CHANGE` metadata even in verbose mode.
- Follow-up diagnostics pass: `SystemServer` warmup remaining now uses the configured `architectWarmupMs` instead of a stale `30_000ms` assumption.
- Follow-up diagnostics pass: `/api/system` and `/api/bots` now expose drawdown-pause/manual-resume state separately from the shared portfolio kill switch.
- Follow-up UI pass: `SystemServer` now serves a separate compact monitor at `/compact`, with optional explicit auto-open through `AUTO_OPEN_COMPACT_UI=true` or `COMPACT_UI=true`.
- Follow-up consistency pass: entry now blocks during pending Architect challenger hysteresis with `architect_challenger_pending`.
- Follow-up consistency pass: managed-recovery invalidation now requires grace/confirmation for early regime-sensitive invalidation, and confirmed recovery target beats invalidation.
- Follow-up economics pass: `rsiReversion` now uses `minExpectedNetEdgePct: 0.0015` plus `maxTargetDistancePctForShortHorizon: 0.01`; entry diagnostics include `targetDistancePct`.
- Follow-up MTF pass: optional MTF frame snapshots, aggregation, and publish diagnostics now use internal horizon frame ids (`short`, `medium`, `long`) while leaving raw timeframe mapping in frame config / aggregation plumbing.
- Follow-up RSI MTF pass: `mtfParamResolver` resolves RSI entry hints/cap diagnostics only. Disabled/absent/unclear/unstable/insufficient/non-range MTF keeps the baseline cap; coherent range `medium` widens to `1.5x`, coherent range `long` widens to `2.0x`, and `short` remains baseline.
- Follow-up telemetry pass: full and compact entry telemetry now exposes published MTF diagnostics plus entry-side RSI cap-resolution fields (`mtfDominantFrame`, `mtfAdjustmentApplied`, resolved cap, target-distance profile, fallback/coherence reason).

## Still Open

- Full modern backtest integration is not done. The repo now has a stable adapter boundary, but replay still runs through legacy logic rather than the modern `ContextService` / `ArchitectService` / `TradingBot` path.
- Short support is still blocked by long-only assumptions in strategy actions, exit target checks, stop logic, chart labeling, and risk semantics.
- There is still no operator-facing manual resume command/API. The runtime now reports the paused/manual-resume state clearly, but resume control remains whatever explicit external mechanism the current runtime uses.
- Compact monitor window sizing is not guaranteed cross-platform. Startup auto-open only asks the OS/browser to open the compact route.
- MTF is enabled by default in `src/data/bots.config.json`; use `MTF_ENABLED=false` to disable it at runtime or `MTF_ENABLED=true` to force it on if JSON config is changed.
- MTF diagnostics are observable, but there is no UI control surface for MTF tuning.
